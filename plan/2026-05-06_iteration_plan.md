## 三、技术债务

### 3.1 错误处理规范化

**创建日期**: 2026-05-06

**问题描述**:
代码中广泛使用 `try?` 静默忽略错误，导致错误信息丢失，难以排查问题。

**问题示例**:
```swift
// 广泛使用 try? 吞掉所有错误信息
let result = try? someFunction()
_ = result
```

**影响范围**:
- SSH 连接错误无感知
- 数据库操作失败不可追溯
- 用户无法获得错误反馈

**改进方案**:
1. 建立统一的 `AppError` 错误枚举，分类处理
2. 关键操作改用 `do-catch` 捕获错误
3. 错误日志持久化到本地
4. 可恢复错误实现重试机制

**涉及模块**:
| 模块 | 文件 | 问题 |
|------|------|------|
| SSH 连接 | SSHClientManager.swift | 连接错误静默失败 |
| SFTP 操作 | RealSSHConnection.swift | 文件传输错误无感知 |
| 数据库 | ServerRepository.swift | CRUD 操作失败不可追溯 |
| 凭据管理 | KeychainHelper.swift | 存取错误被忽略 |

**预估工时**: 3h
**状态**: 待开始

### 3.3 日志框架替换 print

**创建日期**: 2026-05-06

**问题描述**:
散落的 `print()` 语句无法分类控制，无法持久化，无法区分来源。

**问题示例**:
```swift
// 散落的 print 语句
print("Connecting to \(host)")
print("Error: \(error)")
print("Success")
```

**问题影响**:
- 无法控制日志级别（debug/info/warning/error）
- 无法区分日志来源（SSH/Command/Database）
- 无法输出到文件（生产环境无法查看）
- 发布后无法关闭调试日志

**改进方案**:
使用 `os.log` (iOS 14+) 框架，配置 Category 和日志级别。

**日志分类**:

| Category | 用途 | 示例 |
|----------|------|------|
| SSH | SSH 连接相关 | 连接建立、断连、重连 |
| Command | 命令执行相关 | 命令解析、执行结果 |
| Database | 数据库操作 | CRUD 操作、事务 |
| UI | 界面交互 | 视图切换、用户操作 |
| Network | 网络请求 | TCP 连接、数据传输 |

**使用示例**:
```swift
import os.log
private let logger = Logger(subsystem: "com.nexus.shell", category: "SSH")

logger.debug("Connecting to \(host)")
logger.info("Connection established")
logger.error("Connection failed: \(error.localizedDescription)")
```

**预估工时**: 2h
**状态**: 待开始
